# Lexical and Syntax Analyzer with Yacc (Bison)

## Lexical Analyzer (Lex)

### Declarations
```c
%{
#include<stdio.h>
#include "y.tab.h"
extern int yylval;
%}
```
This section includes necessary headers and declares external variables used in the associated Yacc (Bison) file.

### Lexical Rules
```c
%%
[a-zA-Z][a-zA-Z0-9]* {
    yylval = atoi(yytext);
    return IDENTIFIER;
}

[\t]    ;
[\n]    return 0;
.       return yytext[0];
%%
```
- The first rule recognizes identifiers (alphanumeric strings starting with a letter).
- The second rule skips tabs.
- The third rule skips newlines.
- The fourth rule returns any other single character.

### Wrap-Up Function
```c
int yywrap()
{
    return 0;
}
```
This function is called when the end of the input is reached. It returns 0, indicating the end of the input.

## Syntax Analyzer (Yacc/Bison)

### Declarations
```c
%{
#include<stdio.h>
int flag=0;
int yylex();
void yyerror();
%}
```
This section includes necessary headers and declares global variables (`flag`) and functions (`yylex`, `yyerror`) used in the Yacc file.

### Token Declaration
```c
%token IDENTIFIER
```
Declares the token type `IDENTIFIER` to be used in the Yacc rules.

### Syntax Rules
```c
%%
E: IDENTIFIER;
%%
```
Defines a simple syntax rule that an expression (E) can be an identifier.

### Main Function
```c
void main()
{
    printf("\nEnter Identifier: ");
    yyparse();
    if(flag==0)
        printf("\nEntered identifier is Valid\n\n");
}
```
Prompts the user to enter an identifier, then calls `yyparse` to start parsing. If `flag` is 0, it indicates that the entered identifier is valid.

### Error Handling Function
```c
void yyerror()
{
    printf("\nEntered identifier is Invalid\n\n");
    flag=1;
}
```
Prints an error message when a syntax error is encountered during parsing.

## Example Usage
1. Compile the Lex file: `lex expression.l`
2. Compile the Yacc file: `yacc -d expression.y`
3. Compile the C files generated by Lex and Yacc: `gcc lex.yy.c y.tab.c`
4. Run the compiled program: `./a.out`

The program will prompt you to enter an identifier. It will then check if the entered identifier is valid based on the defined syntax rule.

Note: This example is a simple illustration and may need adjustments based on your specific requirements. The error handling and validation are basic and may not cover all cases.
